**SysAnalytiq** — это демонстрационная веб-платформа, разработанная в рамках выпускной квалификационной работы. Она объединяет специализированные инструменты для обучения системному анализу в едином интерфейсе, предоставляя возможности для практики и проверки знаний.

## Возможности платформы

### 1. Пять встроенных редакторов
Платформа предоставляет единую среду для работы с ключевыми нотациями и языками:
*   **SQL**: Редактор кода с подсветкой синтаксиса и возможностью выполнения запросов.
*   **BPMN**: Визуальный редактор для моделирования бизнес-процессов (на базе `bpmn-js`).
*   **ERD**: Визуальный редактор для проектирования схем баз данных (на базе `React Flow`).
*   **PlantUML**: Редактор "диаграммы как код" с мгновенным предпросмотром.
*   **Swagger (OpenAPI)**: Редактор спецификаций API с визуализацией в Swagger UI.

### 2. Режим "Песочница"
Свободный режим, где студенты могут практиковаться в использовании инструментов без привязки к конкретным заданиям.

### 3. Учебные модули и уроки
*   **Выполнение заданий**: Интерфейс для решения практических задач в каждом учебном модуле.
*   **Автоматическая проверка**: Формальная проверка решений модулями проверки (синтаксис, наличие обязательных элементов диаграммы, сравнение результатов SQL-запросов и т. д.).
*   **Ручная проверка**: Отправка решений преподавателю с историей попыток и построчными комментариями.

### 4. Административный интерфейс
*   **Управление учебными материалами**: Создание и редактирование уроков с настройкой критериев автоматической проверки.
*   **Проверка решений**: Просмотр работ студентов, построчные комментарии, принятие работы или отправка на доработку.
*   **ИИ-помощник проверки**: Черновые замечания к решению от языковой модели (через OpenRouter), которые преподаватель может принять, изменить или удалить.

### 5. Уведомления и Telegram-бот
Веб-уведомления о результатах проверки, браузерные push-уведомления и Telegram-бот для оповещений студентов и преподавателей.

## Технологический стек

### Frontend
*   **Framework**: React 18 + TypeScript
*   **Build Tool**: Vite
*   **Styling**: Tailwind CSS + Shadcn UI
*   **State Management**: React Context / Hooks
*   **Libraries**:
    *   `reactflow` (ERD диаграммы)
    *   `bpmn-js` (BPMN диаграммы)
    *   `@monaco-editor/react` (Редактор кода)
    *   `swagger-ui-react` (Визуализация API)

### Backend
*   **Backend**: Python, Django, Django Ninja (Clean Architecture), PostgreSQL.
*   **DevOps**: Docker, Docker Compose.

## Запуск проекта

Проект полностью контейнеризирован. Для запуска требуется **Docker** и **Docker Compose**.

### Локальный запуск

1.  **Создайте `.env` в корне проекта** на основе `.env.example`:
    ```bash
    cp .env.example .env
    ```
    Минимально нужно задать `SECRET_KEY`, параметры PostgreSQL и ключи внешних сервисов:
    VK ID (`VK_APP_ID`, `VK_APP_SECRET`, `VITE_VK_APP_ID`) для авторизации,
    Telegram-бот (`TELEGRAM_BOT_TOKEN`, `VITE_TELEGRAM_BOT_NAME`) для уведомлений и
    OpenRouter (`OPENROUTER_API_KEY`) для ИИ-проверки.

2.  **Авторизация через VK ID требует HTTPS** и не принимает `localhost`. Для локальной
    разработки поднимите туннель через [ngrok](https://ngrok.com/download):
    ```bash
    ngrok http 5173
    ```
    Полученный адрес вида `https://abc123.ngrok-free.app` пропишите в настройках
    VK-приложения (Redirect URI: `https://abc123.ngrok-free.app/login`) и в `.env`
    (`SITE_URL`, `CORS_EXTRA_ORIGINS`).

3.  **Соберите и запустите контейнеры**, примените миграции, загрузите учебные данные и
    создайте администратора:
    ```bash
    make build
    make up
    make migrate
    make seed-all
    make createsuperuser
    ```
    Приложение будет доступно по адресам:
    *   **Frontend**: http://localhost:5173
    *   **Backend API**: http://localhost:8000
    *   **Swagger Docs**: http://localhost:8000/api/docs

4.  **Остановка**:
    ```bash
    make down
    ```

### Загрузка учебных данных (seed)

Команды наполнения базы можно запускать по отдельности — в зависимости от того, что нужно:

| Команда | Что загружает |
|---|---|
| `make seed` | Только учебные модули |
| `make seed-lessons` | Только уроки (модули должны быть уже загружены) |
| `make seed-all` | Модули и уроки (полное наполнение) |
| `make seed-demo` | Демонстрационные решения студентов (для проверки интерфейса) |
